## 📖 Сценарий 1: Студент готовится к экзамену

### Актер: Айдар, студент 2 курса, Computer Science

**Начало:** Понедельник, 18:00. Экзамен по "Data Structures" в пятницу.

```
1. ВХОД (18:05)
   - Айдар открывает Mentoro
   - Логинится в аккаунт
   - Видит красное уведомление "Exam in 4 days!"
   - Кликает на "Data Structures" в календаре

2. ПРОСМОТР МАТЕРИАЛОВ (18:10)
   - Видит список модулей курса
   - Выбирает "Advanced Trees" как сложный участок
   - Нажимает на AI Chat

3. ВЗАИМОДЕЙСТВИЕ С AI (18:15-18:45)
   Айдар: "Объясни Red-Black Trees простыми словами"
   Mentoro AI: "Red-Black Trees - это особый вид...
              Представь как... [диаграмма]...
              Ключевые правила: 1. Каждый узел красный или черный..."

   Айдар: "Зачем нам нужны красные и черные цвета?"
   Mentoro AI: "Отличный вопрос! Цвета помогают балансировать...
              Без баланса дерево становится деградированной линией
              [примеры и диаграммы]"

   Айдар: "Напиши пример кода в Java"
   Mentoro AI: "
   public class RedBlackTree {
       private Node root;

       public void insert(int value) {
           // Implementation
           // 1. Normal BST insert
           // 2. Color adjustment
           // 3. Rotation if needed
       }
   }
   "

4. ГЕНЕРИРОВАНИЕ КВИЗА (19:00)
   - Айдар загружает 3 файла с лекциями
   - Нажимает "Generate Quiz"
   - Выбирает "20 вопросов, Hard level"
   - AI генерирует тест за 30 секунд

5. ПРОХОЖДЕНИЕ КВИЗА (19:30-20:15)
   - Quiz: 20 вопросов, 45 минут
   - Айдар отвечает, получает feedback на каждый
   - Итоги: 16/20 (80%)
   - Слабые места: Rotations, AVL vs RB Trees
   - Mentoro рекомендует: "Review Rotation chapter"

6. ПЛАНИРОВАНИЕ (20:30)
   - Айдар видит в календаре:
     * Сегодня: 2 часа учебы - ✅ Завершено
     * Завтра: Quiz на "Sorting" - плановый
     * Среда: Review session - автоматически добавлено
     * Четверг: Mock exam - предложено AI

7. РЕЗУЛЬТАТ (Пятница)
   - Айдар сдает настоящий экзамен
   - Получает 85/100 (отлично!)
   - Mentoro показывает: "Improvement: +15% vs baseline"
   - Рекомендация: "Great job! Now take next course's preview"
```

### Ключевые моменты ценности:
✅ Сэкономил 5+ часов на поиск объяснений в интернете
✅ Получил персональное объяснение от AI
✅ Автоматически сгенерировал квиз из своих материалов
✅ Понял свои слабые места
✅ Успешно сдал экзамен

## 📚 Сценарий 2: Преподаватель использует Mentoro

### Актер: Айжан, преподавательница курса "Algorithms"

**Начало:** Понедельник, начало семестра

```
1. СОЗДАНИЕ КУРСА (09:00)
   - Айжан логинится в Mentoro Instructor Dashboard
   - Создает курс "Algorithms 2024-Spring"
   - Загружает:
     * Программу курса (syllabus)
     * 12 лекций (PDF)
     * Рекомендованные книги и статьи
     * Требования к заданиям

2. АВТОМАТИЗАЦИЯ ТЕСТОВ (09:30)
   - Вместо ручного создания тестов (обычно 2-3 часа)
   - Используют Mentoro "Auto-Quiz Generator"
   - За 5 минут система генерирует:
     * 10 вопросов для проверки понимания (после каждой лекции)
     * Mock exam из 50 вопросов
     * Adaptive quiz (уровень сложности зависит от ответов)

3. ОТПРАВКА МАТЕРИАЛОВ СТУДЕНТАМ (10:00)
   - Все материалы автоматически загружаются в Mentoro
   - Студенты получают уведомление
   - Календарь автоматически создает напоминания о дедлайнах

4. МОНИТОРИНГ ПРОГРЕССА (В течение недели)
   - Mentoro Dashboard показывает:
     * Какие студенты просмотрели материал: 87% (23/25)
     * Средняя оценка по quiz: 72%
     * Студенты, которые отстают:
       - Марат: 45% (нужна помощь!)
       - Нина: 52% (требует внимания)
     * Самые сложные темы:
       - "Dynamic Programming" - 58% средний результат
       - "Graph Algorithms" - 65% средний результат

5. АДАПТИВНОЕ ПРЕПОДАВАНИЕ (Вторник)
   - На основе данных, Айжан решает:
   - Сегодня на лекции: "Deep dive into DP with examples"
   - Даст дополнительную практику слабым студентам
   - Mentoro помогает найти релевантные проблемы
   - Отправляет "Personalized Learning Path" каждому студенту

6. АВТОМАТИЗИРОВАННАЯ ОБРАТНАЯ СВЯЗЬ (Четверг)
   - Студенты заканчивают Assignment
   - Mentoro автоматически проверяет:
     * Корректность алгоритма
     * Сложность (Big O)
     * Качество кода
   - Дает feedback:
     "Your solution is O(n²) but optimal is O(n log n).
      Try binary search or two pointers approach.
      See this tutorial: [link]"

7. АНАЛИТИКА В КОНЦЕ СЕМЕСТРА (Конец семестра)
   - Айжан видит полный отчет:
     * 87% студентов улучшили оценки на 15%+
     * Самая сложная тема: DP (58%)
     * Рекомендация: увеличить часы на эту тему в следующем семестре
     * Студент-лидер: Марат (+45% улучшение! Отлично!)

8. ОПТИМИЗАЦИЯ КУРСА (Для следующего семестра)
   - На основе данных, обновляет курс:
   - Добавляет больше примеров по DP
   - Переструктурирует "Graph Algorithms"
   - Увеличивает практические задачи
   - Все автоматически кэшируется и готово к началу семестра
```

### Ключевые моменты ценности:
✅ Сэкономила 20+ часов на создание тестов
✅ Получила инсайты, где студенты отстают
✅ Может адаптировать обучение в реальном времени
✅ Видит, какие студенты нуждаются в дополнительной помощи
✅ Данные для улучшения курса в будущем

## 🏫 Сценарий 3: Университет внедряет Mentoro

### Актер: Рахат, Dean of AITU School of CS

**Начало:** Начало года

```
1. ДЕКАНСКИЙ ВСТРЕЧА (09:00)
   - Рахат просматривает результаты:
   - Студенты тратят 3-4 часа в день на учебу
   - Но только 40% материала усваивают эффективно
   - 25% студентов отсеиваются в первый год
   - Есть большие вариации между преподавателями

2. РЕШЕНИЕ: ВНЕДРИТЬ MENTORO (10:00)
   - Заключает контракт с Mentoro: $30K/год для всей школы
   - Включает:
     * Все преподаватели могут создавать AI-powered курсы
     * Все студенты получают "Smart Learning Paths"
     * Analytics dashboard для администрации
     * Technical support и training

3. ПИЛОТ (Месяц 1)
   - 50 студентов в курсе "Data Structures"
   - Сравнительное исследование:
     * Group A (с Mentoro): 15 студентов
     * Group B (без Mentoro): 15 студентов
     * Group C (traditional with TA): 20 студентов
   - Результаты после 4 недель:
     * Group A: Улучшение 23% vs baseline, 91% engagement
     * Group B: +5% улучшение, 67% engagement
     * Group C: Traditonal results, 45% engagement
   - Cost per student for Group A: $2 (vs $50 для TA)

4. МАСШТАБИРОВАНИЕ (Месяцы 2-3)
   - Успех пилота → решение внедрить на все CS программы
   - 500 студентов, 20 преподавателей
   - Все курсы получают:
     * AI-generated quizzes and practice
     * Personalized learning recommendations
     * Real-time progress analytics
     * Adaptive difficulty levels

5. ЦЕНТРАЛЬНАЯ АНАЛИТИКА (Непрерывно)
   - Рахат видит университетскую dashboard:

   Общая статистика:
   - 87% студентов активно используют (vs 40% GitHub)
   - Среднее улучшение оценок: +18%
   - Отсеку снижена на 8% (с 25% до 17%)
   - Среднее время на оговорку: 3.2 часа/день (vs 3.8 раньше)

   Детали по курсам:
   - "Algorithms": +22% улучшение (было самым сложным)
   - "Web Dev": +15% улучшение (практическое применение)
   - "Database Systems": +25% улучшение (strong case!)

   Выявленные проблемы:
   - "Operating Systems": только +8% улучшение
     Рахат: "Что-то не так. Нужно пересмотреть курс"
   - 15 студентов из 500 все еще отстают
     Рахат: "Предложить дополнительный tutoring"

6. ПУБЛИЧНЫЕ ДОСТИЖЕНИЯ (Конец года)
   - AITU объявляет: "Наша программа CS теперь рейтирует выше"
   - Рабочие контакты с Mentoro:
     * Помощь в технологии обучения
     * Персональные траектории для каждого студента
     * Улучшенные результаты экзаменов
   - Это привлекает больше хороших студентов
   - Репутация растет

7. ЭКСПАНСИЯ (Следующий год)
   - AITU решает расширить на все школы (не только CS)
   - Business, Engineering, Architecture все присоединяются
   - 5000 студентов на платформе
   - Mentoro становится центральной частью AITU экосистемы

8. ФИНАНСОВАЯ ВЫГОДА
   - Инвестиция: $30K/год
   - Выгода:
     * +8% студентов с улучшенными оценками = лучше для рейтинга
     * -8% отсева = сбережение $400K (200 студентов × $2K)
     * Лучшая репутация → +20 студентов = +$40K доход
     * Более счастливые студенты → лучше регистрация = +$100K
   - ROI: (400K + 40K + 100K - 30K) / 30K = 14.7x в год 1!

9. ДОЛГОСРОЧНОЕ ВИДЕНИЕ
   - Рахат видит Mentoro как:
     * Ключевая часть AITU стратегии
     * Способ отличиться от других университетов
     * Инструмент для улучшения качества образования
     * Привлечение лучших студентов и преподавателей
```

### Ключевые моменты ценности:
✅ Объективный способ улучшить результаты студентов
✅ Большое сокращение стоимости (TA vs AI)
✅ Данные для принятия решений
✅ Улучшение репутации университета
✅ Положительный ROI в первый год

## 💡 Сценарий 4: Студент в отстающий группе

### Актор: Назар, первокурсник, средний студент

**Начало:** Вторая неделя семестра, Назар чувствует себя потеряным

```
1. ПРОБЛЕМА (18:00)
   - Назар не понимает "Object-Oriented Programming"
   - На лекции было 100 студентов, преподаватель быстро прошел
   - Назар запутался, но боится спросить
   - На TA office hours очередь из 30 человек

2. РЕШЕНИЕ ЧЕРЕЗ MENTORO
   - Открывает Mentoro
   - Видит "Adaptive Learning Path" для OOP
   - Система заметила: "Вы отстаете по OOP" (на основе quiz результатов)
   - Предлагает: "Персональный путь обучения"

3. ПЕРСОНАЛЬНОЕ ОБУЧЕНИЕ (18:15)
   - Mentoro начинает с ОСНОВ (для Назара):

   Шаг 1: "What is a Class?" (5 мин)
   - Видео объяснение (2 мин)
   - Simple example (1 мин)
   - Interactive demo (2 мин)
   - Quiz (1 вопрос) - Назар отвечает правильно

   Шаг 2: "What is an Object?" (5 мин)
   - Similar structure
   - Real-world example: "Dog class" with Nazars own pet
   - Quiz - Назар все еще понимает ✓

   Шаг 3: "Encapsulation" (7 мин)
   - Видео: "Why we hide data"
   - Пример: "Bank account - why password is private?"
   - Код:
     class BankAccount {
         private double balance; // Can't access directly
         public void withdraw(double amount) {
             // Check if enough money
             balance -= amount;
         }
     }
   - Quiz: 2 вопроса - Назар отвечает 1 из 2 правильно
   - AI говорит: "Good! Let's practice more on this concept"

   Шаг 4: Practice Problems (10 мин)
   - 5 практических задач на Encapsulation
   - AI объясняет каждую
   - Назар решает 4 из 5 правильно на первый раз

4. ПРОМЕЖУТОЧНОЕ ЧУВСТВО (19:00)
   - Назар наконец ПОНЯЛ OOP базовые концепты
   - Времени потрачено: 30 минут (vs 2 часа в группе)
   - Уверенность растет: +200%!
   - Никого не стыдился спросить

5. ДОПОЛНИТЕЛЬНАЯ ПОМОЩЬ (19:30)
   - Назар все еще путается с Inheritance
   - Нажимает "Ask AI" в Mentoro

   Назар: "Почему нужна наследование? Почему я просто не сделаю новый класс?"
   Mentoro: "Отличный вопрос! Представь:
   - Есть класс Animal с методами eat(), sleep()
   - Тебе нужны Dog и Cat
   - Без наследования - дублируешь код 2x
   - С наследованием - один раз пишешь, два раза используешь

   Это называется DRY - Don't Repeat Yourself.

   И еще: у собак есть лай, кошки мяукают.
   Наследование позволяет каждому добавить свое поведение!"

   Назар: "Ааааа! Теперь понял!"

6. УСКОРЕНИЕ ОБУЧЕНИЯ (Неделя 2-3)
   - С персональным путем, Назар:
   - Изучает OOP концепты в собственном темпе
   - Получает помощь мгновенно (не ждет TA office hours)
   - Видит прогресс: "86% понимание OOP vs 40% в классе"
   - Выполняет практические задачи с уверенностью
   - Получает помощь в отладке своего кода

7. РЕЗУЛЬТАТ (Конец семестра)
   - Промежуточный экзамен по OOP: 76% (vs прогноз 45%)
   - Финальный проект: B+ (vs ожидаемо C)
   - Тест: 65 → 82% за семестр (+17%)
   - Назар теперь УВЕРЕН в программировании
   - Собирается продолжать CS специализацию

8. ДОЛГОСРОЧНЫЙ ЭФФЕКТ
   - Назар становится лучше
   - В Year 2 он помогает новичкам
   - В Year 4 получает стипендию за хорошие оценки
   - Работает в tech компании после выпуска
   - Обнаруживает Mentoro в LinkedIn - "Это изменило мою жизнь!"
```

### Ключевые моменты ценности:
✅ Личная помощь без стыда
✅ Обучение в собственном темпе
✅ Быстрые результаты (видимый прогресс)
✅ Мотивация растет
✅ Улучшение оценок и уверенности
